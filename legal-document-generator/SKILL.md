# 法律文书生成器

## Overview

根据案情描述智能识别法律文书类型，通过调用子技能（法律文书模版预测技能、法律检索技能）获取模版、法规和类案，生成符合规范的法律文书并输出为docx文档。支持单文书或多文书批量生成。

适用于律师、法务人员需要快速起草各类法律文书的场景。

---

# Process

## High-Level Workflow

生成高质量法律文书涉及四个主要阶段：

### Phase 1: 文书识别与澄清

#### 1.1 识别文书类型
阅读用户输入，识别所需生成的所有法律文书类型。如果识别到多个文书类型，明确告知用户："检测到您需要生成以下文书：[列出所有文书类型]，我将逐一为您生成"。

#### 1.2 澄清反问
根据文书类型提出3-5个关键澄清问题。明确告知用户：
- "请回答上述问题以帮助我更好地生成法律文书"
- "如果您想跳过此环节，请直接回复'跳过'"
- "您也可以只回答部分问题"

**等待用户回复**，绝对不能在用户未回复的情况下继续下一步骤。

**详细指导**: Load [澄清指南](./reference/clarification-guide.md) during this phase.

---

### Phase 2: 模版匹配与检索

#### 2.1 文书模版预测
**调用子技能**：法律文书模版预测技能

**技能位置**：`skills/legal-template-predictor`

**技能说明**：
- 功能：根据用户需求和文书类型，从模版库智能匹配或生成定制模版
- 输入：用户需求、文书类型
- 输出：文书名称、文书简介、模版来源（库内模版/模型生成模版）、模版内容

**调用方式**：使用 load_skill 工具加载该技能，按照其 SKILL.md 中的说明执行。

展示模版信息后，等待用户确认再继续。

#### 2.2 法律法规检索
**调用子技能**：法律检索技能

**技能位置**：`skills/law-retrieval`

**技能说明**：
- 功能：检索与案件相关的法律法规
- 输入：用户描述的法律问题
- 输出：相关的法律法规条文

**调用方式**：使用 load_skill 工具加载该技能，按照其 SKILL.md 中的说明执行。
- 注意：法律检索技能会执行完整的检索流程（问题分析→检索方向→法规查询→法条推荐），按照其SKILL.md中的步骤执行即可。

#### 2.3 类案检索
**调用子技能**：法律检索技能

**技能说明**：
- 功能：检索与案件相关的裁判案例
- 输入：用户描述的法律问题
- 输出：相关的裁判案例

**调用方式**：使用 load_skill 工具加载法律检索技能，按照其 SKILL.md 中的说明执行，检索类案。
- 注意：类案检索也需要执行完整的检索流程，与法律法规检索步骤类似。

---

### Phase 3: 文书生成

#### 3.1 生成文书正文
基于以下信息生成法律文书正文：
- 文书模版（Phase 2.1获取）
- 用户澄清信息
- 检索到的法律法规（Phase 2.2获取）
- 检索到的类案（Phase 2.3获取）

**文书结构要求**：
- 标题
- 当事人信息
- 事实理由
- 请求事项
- 法律依据

**【重要】**：生成文书正文后，**必须立即继续执行Phase 4**，不能停止。

---

### Phase 4: 文档输出

#### 4.1 生成docx文档
**调用脚本**：`scripts/generate_docx.py`

**脚本参数**：
```
--legal-doc-content: 法律文书正文的实际文本内容
--laws-content: 法律法规的实际文本内容
--cases-content: 类案的实际文本内容
--output-path: 输出文件路径（可选，默认为 legal-document.docx）
```

**参数要求**：
- 参数值必须是实际文本内容，**不能是文件路径、参数名、占位符**
- 参数值应该用引号包裹（单引号或双引号）
- 参数值必须是Phase 3生成的文书内容、Phase 2检索的法规和类案内容

**完整流程验证**: Load [流程检查清单](./reference/workflow-checklist.md) before calling the script.
