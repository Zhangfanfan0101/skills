# 输出格式模板

## JSON 输出结构

审查报告输出为 JSON 文件（`.json`），包含以下结构：

```json
{
  "风险点总数": "识别出的风险条款数量",
  "总结与归纳": "整体风险概况和核心发现",
  "谈判策略": "推荐方法和优先级排序",
  "行动建议": "具体采取的行动",
  "风险条款详细分析": [
    {
      "筛选审查内容": "风险条款的具体描述",
      "审查点": "审查要点",
      "风险分析": "风险的具体内容和表现",
      "风险等级说明": "风险等级的判断依据",
      "结论": "不合规",
      "风险等级": "高风险/中风险/低风险",
      "修正建议": {
        "操作": "replace",
        "锚定文本": "用于定位修改位置的文本",
        "原始文本": "需要修改的条款内容",
        "新文本": "修改后的内容",
        "修正说明": "按责任归属分担费用，明确发包人因设计原因导致的超限应自行承担，避免承包人承担非自身过错成本。",
        "参考依据": [
          "《中华人民共和国民法典》第五百零九条：当事人应当按照约定全面履行自己的义务。",
          "《中华人民共和国建筑法》第十八条：建设单位应当提供符合施工要求的场地和条件。"
        ]
      }
    }
  ]
}
```

## 字段说明

### 顶层字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 风险点总数 | Number | 识别出的风险条款数量 | 3 |
| 总结与归纳 | String | 整体风险概况和核心发现 | "高风险：多项条款严重偏离标准，需重点关注费用责任划分和风险承担机制" |
| 谈判策略 | String | 推荐方法和优先级排序 | "首先提出第一级（必须）问题，用第三级（可以）条款交换第二级（应该）问题" |
| 行动建议 | String | 具体采取的行动 | "与法务部门协商费用责任划分条款；准备备选方案；安排与对方律师沟通时间" |

### 风险条款详细分析（数组元素）

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 筛选审查内容 | String | 风险条款的具体描述 | "运输超大件或超重件所需的道路和桥梁临时加固改造费用和其他有关费用由承包人承担。" |
| 审查点 | String | 审查要点 | "超大件运输加固费用由承包人无条件承担" |
| 风险分析 | String | 风险的具体内容和表现 | "该条款未区分超重超限原因是否因发包人设计错误或场地条件导致，强制承包人承担所有相关费用，使承包人承担不可预见、不可控的第三方基础设施改造成本，责任边界模糊，缺乏费用上限或责任归属判断机制。" |
| 风险等级说明 | String | 风险等级的判断依据 | "中等风险，可能导致承包人承担额外成本，需明确责任划分" |
| 结论 | String | 固定为"不合规" | "不合规" |
| 风险等级 | String | 风险等级 | "中风险" |

### 修正建议（对象）

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 操作 | String | 操作类型，固定为"replace"或"delete" | "replace" |
| 锚定文本 | String | 用于定位修改位置的文本 | "编辑栏 超大件或超重件所需的道路和桥梁临时加固改造费用和其他有关费用由承包人承担。" |
| 原始文本 | String | 需要修改的条款内容 | "运输超大件或超重件所需的道路和桥梁临时加固改造费用和其他有关费用由承包人承担。" |
| 新文本 | String | 修改后的内容 | "因发包人提供图纸或设计导致的超大件、超重件运输，其道路和桥梁临时加固改造费用及相关费用由发包人承担；因承包人施工方案或设备选型造成超限的，由承包人承担。" |
| 修正说明 | String | 修改理由和依据 | "按责任归属分担费用，明确发包人因设计原因导致的超限应自行承担，避免承包人承担非自身过错成本。" |
| 参考依据 | Array | 引用的法律法规，字符串数组 | ["《中华人民共和国民法典》第五百零九条", "《中华人民共和国建筑法》第十八条"] |

## 注意事项

- JSON 格式必须正确，所有字符串必须用双引号包裹
- 数组使用方括号 `[]`，对象使用花括号 `{}`
- 中文字段名需保持一致，不得随意修改
- 参考依据数组中的元素必须是字符串
- 操作字段仅支持 "replace" 或 "delete"
- 结论字段固定为"不合规"
- 风险等级字段仅支持 "高风险"、"中风险"、"低风险"
