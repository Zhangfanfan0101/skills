# 合同审计技能

## 技能概述

本技能用于对合同进行专业审查，识别风险条款，提供修正建议和谈判策略。

## 触发条件

当用户出现以下情况时，必须使用此技能：
- 上传合同文件（.docx 或 .pdf）并要求审查
- 要求"审查合同"、"合同审计"、"合同风险分析"
- 要求分析合同条款、识别合同风险
- 要求提供合同修正建议或谈判策略

## 工作流程

### 步骤 1：提取合同内容

使用 `extract_contract.py` 脚本提取合同文本：

```bash
python /app/working/scripts/extract_contract.py <合同文件路径>
```

支持的格式：`.docx`、`.pdf`

### 步骤 2：合同内容审查

按照以下框架对合同进行审查：

#### 2.1 整体性审查原则（参考 audit_principles.md）

- **先整体，后局部**：先确定合同性质、交易目的、主体合法性，再审查具体条款
- **先合法性，后适当性**：先审查合同是否有效、是否存在法律障碍
- **先框架，后细节**：先审查合同结构是否完整，再审查具体表述

#### 2.2 条款审查框架（参考 clause_analysis_framework.md）

对以下 11 类条款进行逐一审查：

| 序号 | 条款类别 | 关键审查点 |
|------|----------|------------|
| 1 | 合同当事人条款 | 当事人名称、主体资格审查、代理权限 |
| 2 | 标的条款 | 标的描述、合法性、可控制性 |
| 3 | 数量条款 | 数量约定、计数方式、调整条件 |
| 4 | 质量标准条款 | 质量标准、质量责任、合法性 |
| 5 | 履行期限、地点和方式条款 | 履行期限、地点、方式、交易方式设计原则 |
| 6 | 价款或报酬条款 | 金额或计算方式、付款条件、方式、对价平衡 |
| 7 | 违约责任条款 | 违约行为界定、违约责任合理性、违约金计算方式、免责条款 |
| 8 | 不可抗力条款 | 不可抗力范围、通知义务、法律后果 |
| 9 | 保密条款 | 保密范围、期限、义务、例外 |
| 10 | 争议解决条款 | 争议解决方式、管辖法院或仲裁机构、适用法律、法律费用承担 |
| 11 | 合同生效条件条款 | 生效条件、生效时间 |

### 步骤 3：风险评估

对每个识别的风险条款进行评估：

| 风险等级 | 判断标准 |
|----------|----------|
| 高风险 | 可能导致合同无效、面临监管处罚、核心资产损失、无限责任 |
| 中风险 | 可能导致额外成本、权利受限、但有补救空间 |
| 低风险 | 不影响合同效力，但存在优化空间 |

### 步骤 4：生成修正建议

为每个风险条款提供：
- **操作类型**：replace（替换）或 delete（删除）
- **锚定文本**：用于定位修改位置的文本
- **原始文本**：需要修改的条款内容
- **新文本**：修改后的内容
- **修正说明**：修改理由和依据
- **参考依据**：引用的法律法规

### 步骤 5：制定谈判策略（参考 negotiation_strategy.md）

按照优先级分类：

| 优先级 | 类型 | 说明 |
|--------|------|------|
| 第一级 | 必须解决 | 核心底线问题，如无上限责任、监管数据保护、知识产权条款冲突 |
| 第二级 | 应该解决 | 重要但可协商，如责任上限调整、赔偿范围、终止灵活性 |
| 第三级 | 可以让步 | 谈判筹码，如适用法律偏好、通知期限 |

### 步骤 6：输出审查报告

按照 output_format_template.md 的 JSON 格式输出审查报告。

## 输出格式

审查报告必须为 JSON 格式，包含以下结构：

```json
{
  "风险点总数": <Number>,
  "总结与归纳": "<String>",
  "谈判策略": "<String>",
  "行动建议": "<String>",
  "风险条款详细分析": [
    {
      "筛选审查内容": "<String>",
      "审查点": "<String>",
      "风险分析": "<String>",
      "风险等级说明": "<String>",
      "结论": "不合规",
      "风险等级": "高风险/中风险/低风险",
      "修正建议": {
        "操作": "replace/delete",
        "锚定文本": "<String>",
        "原始文本": "<String>",
        "新文本": "<String>",
        "修正说明": "<String>",
        "参考依据": ["<String>", ...]
      }
    }
  ]
}
```

## 关键审查要点

### 合同主体合法性
- 审查当事人是否具有相应的民事主体资格
- 审查是否具备从事相关交易的资质或许可
- 审查代理人签署时代理权限是否明确

### 合同标的合法性
- 审查标的是否被法律禁止或限制交易
- 审查标的是否侵犯他人合法权益
- 审查标的是否符合法律法规和行业标准

### 交易行为合法性
- 审查合同交易本身是否违反法律、行政法规的强制性规定
- 审查合同履行期限是否合法（如租赁期限不超过20年）
- 审查具体条款是否违反法律强制性规定

### 违约责任合理性
- 审查违约金是否过高（不超过合同金额的30%）
- 审查违约责任是否对等
- 审查是否存在单方面免责条款

### 争议解决公平性
- 审查管辖法院约定是否公平
- 审查仲裁条款是否合理
- 审查法律适用是否明确

## 注意事项

1. 必须使用 extract_contract.py 提取合同内容
2. 必须按照 output_format_template.md 的格式输出 JSON
3. 风险等级仅支持：高风险、中风险、低风险
4. 结论字段固定为"不合规"
5. 操作字段仅支持：replace、delete
6. 参考依据必须引用具体的法律法规条文
